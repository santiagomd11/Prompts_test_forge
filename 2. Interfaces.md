## WHAT COUNTS AS A PUBLIC INTERFACE (TS/JS)
Scope is limited to **files, classes, methods, and functions** that are **newly introduced and exported** at the module level.

A **public interface** is anything that becomes externally reachable under normal TypeScript/JavaScript module rules ‚Äî meaning it is **exported** from a module, directly or via `export default`.

Publicness is determined **solely by syntax**, not by folder names, comments, or assumptions about internal vs external usage.

The following elements count as newly introduced public interfaces:

* **File** ‚Äî A new source file that exports at least one class or function (named or default).
* **Class** ‚Äî `export class` or `export default class`.
* **Method** ‚Äî A new **non-private** method inside an exported class (not `private`, not `protected`, not `#`).
* **Function** ‚Äî Any exported callable declared at top level, including:
  - `export function f(...)`
  - `export default function f(...)`
  - `export const f = (...) => ...`
  - `export default (...) => ...`
  - `export default async (...) => ...`

---

### ‚ùå DO NOT CONSIDER
Exclude the following, even if exported:
- constants or variables that are not callable  
- TypeScript-only constructs (`type`, `interface`, `enum`)  
- re-exports (`export * from`, `export { ... }`)  
- fields or properties  
- constructor parameters  
- inner functions (nested inside another function or class)  
- anything that is **not** syntactically exported

---

### üö´ DO NOT INFER INTERNAL/EXTERNAL INTENT
Do **not** assume that certain directories (like `internal/`, `lib/`, or `generators/`) imply non-public intent.  
If the code **exports** a function, class, or default callable, it **is** considered a public interface for this analysis.

The only valid reason to exclude an element is **lack of an export statement**, not a path pattern or naming convention.

## OUTPUT FORMAT FOR NEW PUBLIC INTERFACES

Output **only** in this exact structured format**‚Äîone block per interface**:

* path: <relative file path>  
  name: <qualified name>  
  type: <file | class | method | function>  
  inputs: <comma-separated parameters or "None">  
  outputs: <return type or "None">  
  description: <one short sentence>

### Examples

* path: src/router/index.ts  
  name: Router.navigate  
  type: method  
  inputs: path: string  
  outputs: void  
  description: Navigates programmatically to the specified route.

* path: src/utils/parse-config.ts  
  name: parseConfig  
  type: function  
  inputs: input: string  
  outputs: Config  
  description: Parses a configuration string.

* path: src/core/router.ts  
  name: Router  
  type: class  
  inputs: None  
  outputs: None  
  description: Exported router class.

* path: src/api/index.ts  
  name: src/api/index.ts  
  type: file  
  inputs: None  
  outputs: None  
  description: New public entrypoint file exporting API functions.
